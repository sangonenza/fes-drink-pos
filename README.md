# 🍻 ドリンクPOS — ロックフェス ドリンク会計・在庫管理

ブラウザで動くPOSレジ。複数台のスマホで在庫・売上をクラウド同期する。
要件は [requirements.md](requirements.md) を参照。

## 構成

- `index.html` — アプリ本体(依存ライブラリゼロの単一ページ)
- `sw.js` / `manifest.json` / `icon.svg` — PWA対応(ホーム画面追加・圏外リロード耐性)
- `supabase-schema.sql` — Supabase に貼り付けるテーブル定義

同期は Supabase の REST API を素の fetch で叩く方式(6秒間隔のポーリング)。
圏外時は localStorage のキューに溜め、復帰後に自動送信する。会計IDは端末側で
生成したUUID+DB一意制約なので、再送されても二重登録されない。

## セットアップ(当日までに1回)

1. **Supabase プロジェクト作成**(無料枠でOK): https://supabase.com → New project
2. **テーブル作成**: ダッシュボード → SQL Editor → `supabase-schema.sql` の全文を貼り付けて Run
3. **キー取得**: Settings → API Keys から `Project URL` と APIキー(新形式 `sb_publishable_…`、旧ダッシュボードでは `anon public`)をコピー
4. **ホスティング**: このフォルダを Netlify Drop / GitHub Pages / Vercel などに置く
   (HTTPSであればどこでもよい。PWA機能にHTTPSが必要)
5. **端末設定**: スマホでURLを開く → ⚙︎設定 → URL と anon キーを貼り付けて保存 → 接続テスト
6. **他の端末へ配布**: 設定画面の「設定共有リンクをコピー」→ LINE等で共有 → 各端末で一度開くだけ
7. 各端末で「ホーム画面に追加」しておくとアプリのように全画面で使える

## 手元で試す(同期なしのローカルモード)

```sh
cd fes-drink-pos
python3 -m http.server 8123
# → http://localhost:8123 を開く
```

同期未設定でも全機能が動く(ヘッダに「ローカル」と表示)。

## 当日の使い方

- **レジ**: 商品をタップ(複数可)→ 下のカートで数量調整 → 会計へ → 現金/PayPay → 確認 → 決済完了
  - 現金は預かり金額を入力するとお釣りを大きく表示
  - 在庫0の商品は自動で「売切」(生ビールは残数表示なし・売切にもならない)
- **在庫**: ＋補充(ケース単位入力)/ −ロス(廃棄・こぼし・スタッフ利用)
  - 生ビールは**使い捨てカップの実数**で管理: 開始時にカップを数えて「初期搬入」で登録
- **棚卸し**: 在庫画面 → 棚卸しへ。数え終わるまで理論在庫は見えない(ブラインドカウント)。生ビールは残りカップ数を数える → カップ使用数と販売記録の差異が出る
- **集計**: 売上・現金/PayPay別・商品別・時間帯別。日別フィルタ(本日/各日/全期間)付き。会計の取消もここから
- **レジ点検**: 集計画面 → レジ点検へ。実際のレジ金と「本日の」現金売上理論値の差額を記録

## 2日間開催の日次運用

- **1日目の夜**: 棚卸し → レジ点検 → 売上金を回収
- **2日目の朝**: 釣り銭をセットし、レジ点検の「釣り銭準備金」欄にその額を入力して点検
- 在庫・商品は自動で持ち越し。集計は既定で「本日」表示なので2日目は2日目分だけが見える

## 運用ルール(ソフトでは防げないミス対策)

- 商品を渡すのは**決済完了画面を見せてから**
- こぼした・破損したら必ず**ロス登録**
- 棚卸しは全端末が「同期OK」になってから

## セキュリティ上の注意

anon キー+URL がそのままアクセス権になる簡易構成(1日イベント向け)。
設定共有リンクは店のメンバー以外に渡さない。フェスが終わったら Supabase の
プロジェクトを Pause または削除しておくと安全。
